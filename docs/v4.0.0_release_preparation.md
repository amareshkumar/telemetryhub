### Potential Issues & Fixes

#### Issue 1: SerialPortSim Not in day16-clean Branch

**Problem:** If you removed `SerialPortSim.cpp` from the `day16-clean` branch (it's in the list of files you `git rm`'d), but `device/CMakeLists.txt` still references it, the Windows CI build will fail.

**Check:**
```powershell
# On day16-clean branch
git checkout day16-clean
cat device/CMakeLists.txt | Select-String "SerialPortSim"
```

**Fix:** Remove SerialPortSim from CMakeLists.txt on day16-clean:
```cmake
# device/CMakeLists.txt
add_library(device STATIC
    src/FileHandle.cpp    
    src/Device.cpp
    # src/SerialPortSim.cpp  ‚Üê REMOVE THIS LINE on day16-clean
)
```

#### Issue 2: Device.h/Device.cpp References Serial Bus

**Problem:** If `Device.h` has forward declarations for `IBus` or `Device.cpp` has serial command processing, but SerialPortSim is removed, linker errors will occur.

**Check:**
```powershell
grep -n "serial_bus\|IBus\|process_serial" device/include/telemetryhub/device/Device.h
grep -n "serial_bus\|IBus\|process_serial" device/src/Device.cpp
```

**Fix Options:**

**Option A (Recommended): Keep Serial Features in Public Release**
- SerialPortSim is a valuable feature showing hardware abstraction
- Demonstrates SOLID principles (DIP, ISP)
- Shows modern C++ design patterns
- **Action:** Keep SerialPortSim in day16-clean, remove only interview-specific docs

**Option B: Remove Serial Features from Public Release**
- Remove SerialPortSim.cpp from CMakeLists.txt
- Remove `set_serial_bus()` and `process_serial_commands()` from Device.h/Device.cpp
- Remove I2CBus.h, SPIBus.h headers
- Update README to not mention serial/UART features
- **Action:** Significant refactoring required

#### Issue 3: New Test Files Reference SerialPortSim

**Problem:** If `test_serial_port_sim.cpp` exists on day16-clean but SerialPortSim is removed, tests won't compile.

**Check:**
```powershell
git checkout day16-clean
ls tests/test_serial_port_sim.cpp
cat tests/CMakeLists.txt | Select-String "serial"
```

**Fix:** Remove test file:
```cmake
# tests/CMakeLists.txt
# Remove this line:
# add_executable(test_serial_port_sim test_serial_port_sim.cpp)
```

#### Issue 4: CMake Version Mismatch

**Problem:** CMakeLists.txt shows `VERSION 3.0.0`, but you're releasing 4.0.0.

**Fix:** Update version BEFORE merging to main:
```cmake
# CMakeLists.txt line 4
project(TelemetryHub 
    VERSION 4.0.0  # ‚Üê Change from 3.0.0
    LANGUAGES CXX
)
```

## Task 2: Version 4.0.0 Release Process

### What Makes This a Major Version (4.0.0)?

**Major Breaking Changes:**
- New hardware abstraction layer (IBus interface)
- Device API extended with `set_serial_bus()`, `process_serial_commands()`
- Command-based device control (breaking change from simple start/stop)
- Testing framework change (affects downstream developers)

**Why Not 3.1.0?**
- API changes are significant enough to warrant major version bump
- Existing integrations using Device class will need updates
- Follows semantic versioning: MAJOR.MINOR.PATCH

### Version Number Updates

**File 1: CMakeLists.txt**
```cmake
project(TelemetryHub
```
